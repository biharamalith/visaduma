# Firebase Push Notifications Setup Checklist

Quick reference checklist for setting up Firebase Cloud Messaging in VisaDuma.

## Prerequisites
- [ ] Google account with access to Firebase Console
- [ ] Flutter development environment set up
- [ ] Android Studio and/or Xcode installed

## Firebase Console Setup
- [ ] Create Firebase project at [console.firebase.google.com](https://console.firebase.google.com/)
- [ ] Enable Cloud Messaging in Firebase Console
- [ ] Add Android app to Firebase project
- [ ] Add iOS app to Firebase project

## CLI Tools Installation
- [ ] Install Firebase CLI: `npm install -g firebase-tools`
- [ ] Login to Firebase: `firebase login`
- [ ] Install FlutterFire CLI: `dart pub global activate flutterfire_cli`
- [ ] Run FlutterFire configure: `flutterfire configure`

## Dependencies
- [ ] Add `firebase_core: ^3.8.1` to pubspec.yaml
- [ ] Verify `firebase_messaging: ^15.1.5` is present
- [ ] Verify `flutter_local_notifications: ^17.2.1+2` is present
- [ ] Run `flutter pub get`

## Android Configuration
- [ ] Download `google-services.json` from Firebase Console
- [ ] Place `google-services.json` in `android/app/` directory
- [ ] Add Google services plugin to `android/build.gradle.kts`
- [ ] Apply plugin in `android/app/build.gradle.kts`
- [ ] Add permissions to `AndroidManifest.xml`:
  - [ ] `INTERNET`
  - [ ] `POST_NOTIFICATIONS`
  - [ ] `VIBRATE`
  - [ ] `WAKE_LOCK`
- [ ] Add Firebase Messaging Service to `AndroidManifest.xml`
- [ ] Add default notification channel metadata

## iOS Configuration
- [ ] Download `GoogleService-Info.plist` from Firebase Console
- [ ] Add `GoogleService-Info.plist` to Xcode project (Runner folder)
- [ ] Enable Push Notifications capability in Xcode
- [ ] Enable Background Modes capability in Xcode:
  - [ ] Remote notifications
  - [ ] Background fetch
- [ ] Update `AppDelegate.swift` with Firebase initialization
- [ ] Create APNs key in Apple Developer Portal
- [ ] Upload APNs key to Firebase Console (Cloud Messaging settings)

## Code Integration
- [ ] Verify `lib/core/services/notification_service.dart` exists
- [ ] Update `lib/main.dart` to initialize Firebase
- [ ] Uncomment Firebase initialization in `_initializeFirebase()`
- [ ] Import `firebase_options.dart` in main.dart

## Testing
- [ ] Run app on Android device/emulator
- [ ] Run app on iOS device/simulator
- [ ] Check debug console for FCM token
- [ ] Verify "NotificationService initialized successfully" message
- [ ] Send test notification from Firebase Console
- [ ] Test foreground notification display
- [ ] Test background notification delivery
- [ ] Test notification tap handling

## Backend Integration
- [ ] Create API endpoint for FCM token registration
- [ ] Implement token storage in database (fcm_tokens table)
- [ ] Set up Firebase Admin SDK on backend
- [ ] Implement notification sending logic
- [ ] Test end-to-end notification flow

## Security
- [ ] Add Firebase config files to `.gitignore`
- [ ] Use separate Firebase projects for dev/prod
- [ ] Implement server-side token validation
- [ ] Set up rate limiting for notifications

## Production Readiness
- [ ] Test on multiple Android devices
- [ ] Test on multiple iOS devices
- [ ] Verify all notification channels work correctly
- [ ] Test notification preferences
- [ ] Implement analytics for notification delivery
- [ ] Set up monitoring and alerting
- [ ] Document notification payload formats
- [ ] Create runbook for common issues

- ✅ `FIREBASE_CHECKLIST.md`
- ⬜ `lib/firebase_options.dart` (generated by flutterfire configure)

### Files to Modify
- ✅ `lib/main.dart` (add Firebase initialization)
- ⬜ `android/app/build.gradle.kts` (add Google services plugin)
- ⬜ `android/build.gradle.kts` (add classpath)
- ⬜ `android/app/src/main/AndroidManifest.xml` (add permissions and services)
- ⬜ `ios/Runner/AppDelegate.swift` (add Firebase initialization)
- ⬜ `pubspec.yaml` (add firebase_core dependency)
- ⬜ `.gitignore` (add Firebase config files)

### Files to Download
- ⬜ `android/app/google-services.json` (from Firebase Console)
- ⬜ `ios/Runner/GoogleService-Info.plist` (from Firebase Console)

## Common Issues

### Android
- **Build fails**: Check google-services plugin is applied correctly
- **No notifications**: Verify google-services.json package name matches
- **Token null**: Check internet permission and Firebase initialization

### iOS
- **No notifications**: Verify APNs key is uploaded to Firebase
- **Build fails**: Check GoogleService-Info.plist is in Xcode project
- **Background not working**: Enable Background Modes capability
